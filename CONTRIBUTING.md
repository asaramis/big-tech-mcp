# Contributing

Thank you for your interest in contributing to the Big Technology Podcast MCP server!

**Note:** This project includes podcast transcripts with permission (Ranjan Roy is co-host).

## What You Can Contribute

✅ **Code improvements** - Bug fixes, features, optimizations
✅ **Documentation** - Better examples, clearer instructions
✅ **Search enhancements** - Better indexing, snippet extraction
✅ **Deployment guides** - New platforms, configuration examples
✅ **Bug reports** - Issues you've encountered
✅ **Feature requests** - Ideas for improvements
✅ **Transcript updates** - New episodes or corrections (via PR)

## What You CANNOT Contribute

❌ **Unauthorized content** - Content from other podcasts without permission
❌ **API keys or credentials** - Keep secrets secret
❌ **Malicious code** - Security is important

## Guidelines

### Code Contributions

1. Fork the repository
2. Create a feature branch
3. Make your changes (code only, no transcripts)
4. Test thoroughly
5. Submit a pull request

### What to Test

When submitting PRs, please test:
- The server starts without errors
- Search works with example data
- TypeScript compiles cleanly
- Documentation is clear

### Code Style

- TypeScript with strict type checking
- Meaningful variable names
- Comments for complex logic
- Follow existing code patterns

## Legal Requirements

By contributing, you agree that:
- Your contributions are your own work or properly attributed
- Your code will be released under the MIT License
- You respect the intellectual property of content creators

## Questions?

Open an issue for discussion before starting major work.
